# SeaForge Palantir Identifiers

This is the checklist of API names and RIDs we need from Foundry / Ontology Manager / Developer Console.

## Required First

- `FOUNDRY_BASE_URL` — stack URL.
- `FOUNDRY_TOKEN` — long-lived scoped token or OAuth access token.
- `FOUNDRY_ONTOLOGY_API_NAME` or `FOUNDRY_ONTOLOGY_RID` — REST v2 accepts either for `{ontology}`.
- Developer Console app resources:
  - Ontology SDK resources: every object type, action, and function/query SeaForge calls.
  - Platform SDK resources: project access plus dataset/stream API operations if using dataset writes.

## Ontology Object Type API Names

REST object and link endpoints use object type API names, not display names.

- `vessel`
- `track`
- `observation`
- `aisMessage`
- `anomaly`
- `claim`
- `evidenceItem`
- `areaOfInterest`
- `collectionAction`
- `reviewRule`
- `operatorDecision`
- `sourceDocument`

Use the actual API names generated by Ontology Manager. The names above are the preferred SeaForge targets, not guaranteed Foundry values.

## Link Type API Names

- `vesselHasTrack`
- `trackHasObservation`
- `observationDerivedFromAis`
- `observationDerivedFromEvidence`
- `anomalyInvolvesTrack`
- `anomalyInvolvesVessel`
- `anomalyInAoi`
- `claimExplainsAnomaly`
- `supports`
- `weakens`
- `contradicts`
- `sourceDocumentContainsEvidence`
- `collectionActionAddressesAnomaly`
- `collectionActionTestsClaim`
- `decisionOnClaim`
- `decisionOnCollectionAction`
- `reviewRuleScopedToAoi`
- `reviewRuleAppliedToAnomaly`
- `appliesTo`
- `reviewedBy`
- `derivedFrom`

## Action Type API Names

Apply-action uses action API names. Action RIDs are useful for audit/debugging but are not the main apply path.

- `saveOperatorDecision`
- `saveReviewRule`
- `requestCollection`
- `updateClaimStatus`
- `mergeVesselIdentity`

## AIP Logic / Query API Names

AIP Logic functions exposed through the API Gateway are query types. Execution uses `queryApiName`.

- `seaforgeKinematicsSpecialist`
- `seaforgeIdentitySpecialist`
- `seaforgeIntentSpecialist`
- `seaforgeCollectionSpecialist`
- `seaforgeVisualSpecialist`
- `seaforgeProvenanceSummary`
- `seaforgeReviewMemoryDraftRule`

Also record each query/function RID and version from Ontology Manager for smoke checks and support conversations.

## Dataset / Stream RIDs

Dataset APIs use dataset RIDs. Record these once created:

- `raw_ais_marinecadastre_daily`
- `raw_ais_aishub_poll_batches`
- `stream_ais_live`
- `raw_danti_results`
- `raw_exa_results`
- `stream_edge_observation`
- `curated_ais_message`
- `curated_observation`
- `curated_vessel_identity`
- `curated_tracklet`
- `curated_track_association_candidate`
- `curated_anomaly`
- `curated_hypothesis_feature`
- `curated_evidence_item`
- `aip_specialist_outputs`
- `action_operator_decision_edits`
- `action_review_rule_edits`

If creating datasets programmatically, also capture the parent folder RID.

## Optional SDK Metadata

Some Ontology v2 action/query endpoints accept generated SDK metadata:

- `FOUNDRY_SDK_PACKAGE_RID`
- `FOUNDRY_SDK_VERSION`

Keep these optional unless the generated OSDK path requires them.
