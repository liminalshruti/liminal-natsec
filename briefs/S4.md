## Item to ship

Cut 08 had a dedicated "alert"-toned banner above the command terminal
showing the live causal trace ("Intent refused because Signal Integrity
contested..."). Our app: the causal trace lives in SpecialistReads
connectors and ExecSummary. Visible — but not in a single pinned
location the operator's eye can return to.

Add a `<CausalBanner>` component that mounts BETWEEN the WorkingPanel
and the CommandLine in `AppShell.tsx`. Renders a single-line live
status: "currently: Intent refused — Signal Integrity contested
(Layer 2)" or "currently: prior rule R-001 applied" or "watchfloor
at rest" depending on scenario state.

**Visual register:**
- Single horizontal band, mono register, ~32px tall
- Tone-coded by current state:
  - REFUSED present in any specialist → `--color-refused` left border + amber tone
  - Rule applied → `--color-resolved` left border + green tone
  - Otherwise → neutral `--color-edge` border, ink-secondary text
- Always-visible, always single-line
- Right-aligned timestamp showing when the current state was reached

The CausalBanner reads from `scenarioState` and `selectedAlert` to
derive the current causal sentence. Logic:
- If selected case has Intent REFUSED → "Intent refused — Signal
  Integrity contested (Layer 2)"
- If selected case has rule applied → "Prior rule R-001 applied —
  recommendation changed from MONITOR to REQUEST_SAR_RF"
- If no case selected → "Watchfloor at rest — 2 contested cases
  pending review"
- During phase transitions → "Phase N — <phase name>"
